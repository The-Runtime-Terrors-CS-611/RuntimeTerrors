User Stories

Chris Soard User Stories

Scenario #1

As an avid gamer, I need a logbook to keep track of all the characters I have met or heard of, and my relationship with them.
As an avid gamer, I want a logbook that shows the places I have been and other locations that I can go to to keep track of what I have done and where I can go from here.
As an avid gamer, I need a logbook that keeps track of any world-building lore that I have learned so I can remember key information when making decisions later in the game.
As an avid gamer, I want a logbook of characters and locations to show the ones I haven’t interacted with yet, which inspires me to explore more of the game.
As an avid gamer, I want to be able to see a decision tree of the decisions I’ve made, so I can get a sense of the character I am and continue to make choices along the same route.

Scenario #2

As an avid gamer, I want to be able to pick up and carry items from the world around me to use in the game.
As an avid gamer, I want to be able to carry food, water, weapons etc. that are necessary for survival, to make the game feel more immersive.
As an avid gamer, I want certain objects to progress story lines or dialogue options that are otherwise unavailable.
As an avid gamer, I want this sack to have a weight limit so that I have to make choices on what to carry and what to leave behind.
As an avid gamer, I want to encounter merchants that I can barter with for quest items or survival needs.

Scenario #3

As an avid gamer, I want certain important decisions to have a timer on them to simulate urgency while playing the game.
As an avid gamer, I want the timer to result in my character making either no choice or a bad choice.
As an avid gamer, I want a timing system that lets me check the time and “rest” until a different time of day that changes the outcome of my actions.
As an avid gamer, I want an in-game time limit (i.e. one week) on finishing the game and saving the villagers, thus creating an even greater sense of urgency and giving each decision more weight.
As an avid gamer, I want a tracker that tells me how much time/many days I have left to solve what happened and save the villagers.

Alex Carter User Stories

Scenario #1

As a deployed mobile CYOA player, I want the game to be fully playable offline so that I can play without reliable internet access.
As a deployed mobile CYOA player, I want my saves to be stored locally on my device so that my progress is available even when I’m offline.
As a deployed mobile CYOA player, I want the game to restore my current scene after an interruption so that I can pick up where I left off.
As a deployed mobile CYOA player, I want a “Return to last branch point” option when a branch ends so that I can explore other choices without restarting the entire story.
As a deployed mobile CYOA player, I want clear messaging when a branch has ended (and why) so that I understand the consequence of my earlier choices.

Scenario #2

As a deployed mobile CYOA player, I want a relationship tracker for key characters so that I can see how my choices are affecting my standing with them.
As a deployed mobile CYOA player, I want certain dialogue or action options to unlock based on relationship levels so that relationships meaningfully change what I can do.
As a deployed mobile CYOA player, I want NPC reactions to reflect my past choices so that the world feels consistent and responsive.
As a deployed mobile CYOA player, I want the game to record major relationship-shifting decisions in a log so that I can understand what led to a specific outcome.
As a deployed mobile CYOA player, I want endings to summarize which relationships helped or harmed my outcome so that replays feel informative and intentional.

Scenario #3

As a deployed mobile CYOA player, I want my resources (like rations or supplies) to be tracked persistently so that survival choices have lasting consequences.
As a deployed mobile CYOA player, I want choices to include clear tradeoffs (gain now, lose later) so that decisions feel grounded and strategic.
As a deployed mobile CYOA player, I want certain options to become unavailable when resources are low so that the game reflects the reality of my earlier choices.
As a deployed mobile CYOA player, I want tactical choices (approach/roles/plan) to change encounter outcomes so that I feel like I’m directing the action rather than watching it.
As a deployed mobile CYOA player, I want failures to lead to fair branch endings with checkpoint recovery so that risk feels meaningful without forcing a full restart.

Scenario #4

As a deployed mobile CYOA player, I want a “Continue” button that returns me to my last checkpoint so that I can jump back in quickly during short breaks.
As a deployed mobile CYOA player, I want checkpoints to occur at natural story beats so that I can stop without losing meaningful progress.
As a deployed mobile CYOA player, I want a brief “Previously” recap when I resume so that I don’t have to reread earlier sections to remember what I chose.
As a deployed mobile CYOA player, I want to save at checkpoints without saving on every screen so that saving feels fair but not abusable.
As a deployed mobile CYOA player, I want the game to autosave at checkpoints so that I don’t lose progress if I close the app quickly.

Julia Finn User Stories

Scenario #1

As a player who plays in short sessions after work, I want the game to automatically save my progress after every meaningful choice, so I can resume exactly where I left off without losing progress.
As a player returning to the game after a break, I want the game to reopen at my most recent decision point, so I can smoothly continue the story instead of replaying large sections.
As a player who cares about narrative consistency, I want a summary or recap of my recent choices when I return, so I can remember what I did and why the story is in its current state.
As a player who sometimes has limited time or energy, I want confirmation that my progress has been saved before I exit the game, so I feel confident closing the app without worrying about losing my place.
As a player who becomes frustrated by replaying content unintentionally, I want the game to track and restore my exact choice history, so the story continues along the same path I previously experienced.

Scenario #2

As a teacher using a CYOA game to teach narrative structure, I want player choices to significantly alter the story’s plot and outcome, so students can experience and compare genuinely different narrative arcs.
As a teacher assigning the game as homework, I want multiple distinct story paths that map to different climaxes and resolutions, so student work reflects diverse interpretations of the same narrative framework.
As a student player, I want my choices to meaningfully influence events and character outcomes, so the story I create feels unique and reflects my decisions.
As a teacher facilitating group discussion, I want the game to clearly highlight major branching points in the story, so students can identify and explain how different choices led to different narrative arcs.
As an educator assessing learning outcomes, I want to be able to see how different choices affect exposition, conflict, and resolution, so I can better connect the game to narrative-writing concepts.

Scenario #3

As a player who uses story games to unwind after work, I want the game to give me a short pause or transition before presenting major choices, so I have time to process the story emotionally and mentally instead of feeling rushed.
As a player who enjoys thoughtful storytelling, I want to be able to reread recent sections of the narrative before making a decision, so I can fully understand the context and implications of my choices.
As a player who experiments with dialogue and crafting options, I want clear indicators of when a choice is especially impactful or irreversible, so I can slow down and reflect when it really matters.
As a player who sometimes makes choices late at night when I’m tired, I want the ability to pause the game at decision points, so I can step away or think without pressure and come back when I’m ready.
As a player who values immersion over challenge, I want a way to undo or revise a recent choice, so an unclear or accidental decision doesn’t derail the story into a direction I didn’t intend.

Kofi Asirifi User Stories

Scenario #1

As a Cybersecurity Manager, I want the game to function entirely offline so I can play securely within my office "think pod" or on airplanes without compromising my data or needing a stable connection.
As a handheld gaming enthusiast, I want local save storage on my device so that my progress remains accessible even when I’m traveling through regions with limited service.
As a mobile gamer with four kids, I want the game to autosave on every screen so I can instantly close the app when a child needs me without losing a single line of progress.
As a tech professional, I want an encrypted cloud-sync option for my saves so I can transition seamlessly from my phone to a dedicated handheld console when I get home.

Scenario #2

As a manager experienced in complex systems, I want a detailed relationship log that shows the specific security or political consequences of my dialogue choices with NPCs.
As a knowledgeable gamer, I want tactical branching that rewards logic and resource management, allowing me to use "tech-like" skills (lockpicking, strategy, or hacking) to bypass obstacles.
As a strategic player, I want a "Branch Review" tool that lets me see the logic path of my previous decisions so I can understand exactly how I reached a specific "dark turn" or success.
As a traveler, I want localized story summaries that remind me of previous plot points after I’ve spent a few days away from the game during a busy trip.

Scenario #3

As a busy father of four, I want a “Previously On” recap every time I resume play so I can instantly regain my immersion during a five-minute bathroom break.
As a user with limited free time, I want clear checkpoint indicators at natural story beats so I know exactly how much time I need to commit to finish a chapter.
As a handheld player, I want a "Quick Rewind" feature that allows me to undo the last three dialogue choices if an interruption caused a misclick or misunderstanding.
As a knowledgeable tech user, I want adjustable text speeds and skip toggles for reread sections so I can efficiently navigate back to where my story took a wrong turn.

Scenario #4

As a Cybersecurity Manager used to risk assessment, I want persistent resource tracking (like gold or gear) where the trade-offs are clearly explained, making my tactical choices feel grounded and logical.
As a strategic thinker, I want branch endings to provide a "post-mortem" summary explaining which specific security or relationship failures led to my outcome.
As a mature gamer, I want high-consequence choices that require significant reflection, paired with a "Think Timer" pause so the game doesn’t force a decision while I’m distracted by my environment.
